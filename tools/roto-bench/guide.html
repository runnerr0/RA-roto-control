RA Roto — Operator Guide

Control console for the Roboteq HDC2450 motor controller. Read the safety section before driving a motor.

1 · What this is

The full system bridges show-control protocols (DMX512, Art-Net, sACN) to a rotating fixture: DMX/Art-Net/sACN → ESP32-POE-ISO → RS232 serial (!G) → Roboteq HDC2450 → motor. Everything drives the motor over serial — one signed command sets speed and direction, the controller's own watchdog self-stops on comms loss, and it streams back real telemetry. No DAC, op-amp, or calibration.

This console is the host-side control surface that drives that serial link today, over USB — for bench bring-up, tuning, and running a self-contained piece. The ESP32 firmware (a follow-up, pending parts) will be the networked front-end feeding the same serial path.

2 · Quick start

  1. Power the motor from a supply that can deliver the current you need. During first bring-up, a current-limited PSU (e.g. 5 A) is a physical safety net.
  2. Keep the controller's hardware E-STOP reachable.
  3. Pick a run mode → press ARM → drive. Stop = release the slider (JOG), press Zero, or E-STOP.

The console starts DISARMED and always logs to logs/. Nothing moves until you ARM.

3 · Safety model

Protection is layered — fastest/most-local first. A fault stops the motor, it doesn't run it.

LayerWhat it does
PSU current limitPhysical, reset-proof. The real ceiling on energy for bring-up.
ALIM foldbackController caps motor current at the set limit.
^RWD watchdogThe console streams !G continuously; if it dies, the controller self-stops in ~0.5 s.
Overtorque backoffWhen pinned at the current limit, gently ramps the command down to relieve torque — so a jam can't wind up and break free violently. (See below.)
Stall / overcurrent tripAmps held near the limit too long → stop + disarm. RUN modes only.
Temperature tripController heatsink temp too high → stop. Note: this is the controller, not the motor.
I²t tripLeaky current²·time estimate of motor heating → stop. Our only motor-winding thermal proxy.
Control-drift alarmWatches ALIM/RWD/AINA vs an expected baseline; alarms if a controller reset reverts them. One-click Re-apply safe profile.
Operator layerArm gate, hold-to-run, browser deadman, command cap + slew, E-STOP.

On any trip: command → 0, motor disarmed, latched until you clear it. A red banner, an audible alarm, and a timestamped alert appear. The command card also shows why output is held at 0 (DISARMED / DEADMAN / TRIPPED / E-STOP) — so "armed but not moving" is never a mystery.

Jam vs. hold: open-loop (no encoder) can't tell a jam from a loaded hold by current alone. That's why RUN modes stall-trip on current while HOLD/CREEP rely on temperature + I²t. An encoder (closed loop) is the real fix.

4 · Run modes

ModeBehaviourUse for
JOGHold the slider to run; release = stop. Deadman-protected.Setup, positioning, testing.
CRUISESet a speed; it holds. Stall-protected.Steady running.
DRIFTAuto slow sweep around a center point (amplitude · center · period · waveform).Ambient / atmospheric motion.
HOLDMaintain position under load; stall-trip off, temp/I²t still guard.Holding against a load.
CREEPVery slow under load; stall-trip off, anti-stiction kick on start.Slow loaded creep.

DRIFT patterns

command = center + amplitude · wave(t). center biases the sweep (0 = balanced fwd/rev; +300 = mostly-forward breathing). waveform: sine (smooth), triangle (linear), saw (ramp+snap). Keep the speed cap ≥ |center| + amplitude or the peaks clip.

5 · Settings & limits

Host-side (console)

Speed capCeiling on command magnitude (0–1000). Limits top speed/duty regardless of mode. If amps won't rise, this is usually why — raise it.
Stall tripAmps ≥ this for this long (RUN modes) → stop + disarm.
Temp / I²tTemperature trip (controller) and the I²t motor-heat budget (A²·s).
Overtorque backoffAt/above this amps for this long → ramp command down to relieve torque. Suppressed during the lift test.
CREEP kickBreakaway boost (level · ms) on start-from-stop to overcome static friction.

Controller config (Advanced profile editor — writes controller RAM, verified)

ALIMCurrent limit (0.1 A → 50 = 5.0 A). Foldback holds motor amps at/below this. Current = torque.
MXPF / MXPRMax power forward/reverse (%). Ceiling on duty = top speed.
RWDCommand-loss watchdog (ms). Serial silence this long → motor stops.
MAC / MDECAccel/decel ramps (ms for 0→100 %). Longer = gentler start (less inrush) and softer stop (less regen spike).
OVL / UVLOver/under-voltage cutoffs (0.1 V). A descending load regenerates and raises bus voltage — a PSU (unlike a battery) can't absorb it. Heavy-lift current sags the supply — set UVL below your worst sag.
ATRIG/ATGA/ATGDController-side amps-trigger: level (0.1 A, below ALIM), action (17 = safety-stop M1), delay (ms). A hardware backstop that survives a host bug.
Profile edits are RAM-only until you Save to flash. A controller reset reverts RAM to flash — the control-drift alarm catches that.

6 · Tools

Characterization sweep

Steps the command up in increments, records the settled amps at each level → a command→amps curve, an estimated breakaway command, and a suggested CREEP kick. Requires ARM; aborts on trip/disarm.

Guided lift test

Holds a slow command and raises ALIM rung by rung until the motor breaks free (amps drop below the limit), then reports the load current and suggests ALIM ≈ peak×1.3. Works in either direction. Requires ARM; aborts on trip/disarm/over-temp; restores the safe limit on any non-success stop. The hold command must be high enough to drive current up to the limit — if it never reaches the limit, raise the hold cmd (and the speed cap).

Logging

Always on by default. Full telemetry + every config change, test event, trip, and E-STOP → a timestamped CSV in logs/. Selectable rate (5 Hz / 15 Hz aggressive).

Advanced graph

Overlay any telemetry (motor A, supply A, power, volts, temp, speed) over time; click a legend chip to toggle a trace.

7 · Tuning a loaded roto

An unbalanced (lifting) load draws current that varies with angle — highest while lifting, low/regenerative on the way down. Tune for the peak-lift point.

  1. Be on the real supply; fuse & wire for the current.
  2. Turn logging to 15 Hz.
  3. Run the guided lift test → find peak-lift current.
  4. Set ALIM ≈ peak × 1.3 in the profile editor.
  5. Run it loaded in CREEP and watch Max temp — stabilising = good, climbing = over the motor's continuous rating.
  6. Set the guards (stall above peak-lift, temp, I²t, CREEP kick) and Save to flash.
If it hits the ceiling without lifting, or temp keeps climbing, the answer is more gear reduction or a run/rest duty cycle — not just more current.

8 · Reading the telemetry

Motor ampsWinding current = torque. The AT-LIMIT badge = the controller is folding back power to hold the limit.
Supply ampsCurrent from the supply. Negative = regen (a descending load).
Power outActual PWM duty applied. If it's less than your command, foldback is active. If it equals your command, you're not current-limited — a low command is the constraint.
StatusActive command mode (serial), plus STALL, AT-LIMIT, pwr-off flags.
Speed0 until an encoder is fitted (open-loop has no motion feedback).

9 · Troubleshooting

Armed but not movingRead the red gate line under the command: DISARMED / DEADMAN / TRIPPED / E-STOP. In JOG use CRUISE for set-and-hold.
Amps won't rise / won't liftPower out = command & no AT-LIMIT → the speed cap is clamping the command. Raise it so amps climb to ALIM.
I²t trip too soonBudget too small for a beefy motor — raise it, keep it on (it's your motor-heat proxy), tune by feeling the motor.
CONTROL DRIFT alarmA controller reset reverted RAM config → Re-apply safe profile, then Save to flash.
Violent break-free fearedKeep ALIM snug (peak×1.3), enable overtorque backoff, gentle MDEC. An encoder makes this precise.
